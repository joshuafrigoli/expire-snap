‚óŹ Real push = browser closed ‚Üí notification still arrives. Web standard requires
  a server to trigger it. No way around this.

  Why: Push notifications flow is:
  Your server ‚Üí Browser Push Service (FCM/Mozilla) ‚Üí user device

‚ĚĮ why do i lose HTML mockups?                                                   

‚óŹ "Lose" was too strong. Clarifying:

  Still useful: mockups as visual reference ‚ÄĒ colors, layout, spacing, component
  shapes. You can look at code.html and replicate the same design in React
  Native components.

  Can't copy-paste: HTML structure and CSS classes. Every <div> becomes <View>,
  <p> ‚Üí <Text>, <button> ‚Üí <Pressable>, no className, no raw CSS.

  NativeWind closes most of the gap ‚ÄĒ Tailwind utility classes work on RN
  components, so token names like bg-primary, rounded-full, shadow-hard
  translate directly. The design system tokens survive the port mostly intact.

  What actually breaks: CSS features RN doesn't support ‚ÄĒ box-shadow needs
  elevation on Android, pseudo-elements (::before/::after) don't exist,
  overflow: hidden has quirks, position: fixed doesn't exist.

  So: mockups stay valid as design spec. HTML structure becomes useless as code.
  NativeWind handles ~80% of styling. The hard-shadow neo-brutalism style
  specifically may need manual workarounds since box-shadow with offset is
  web-only ‚ÄĒ RN shadow system is more limited.

  1. Need Apple Developer account ($99/year)
  2. eas build --platform ios ‚Üí generates .ipa
  3. Upload via eas submit or Transporter to App Store Connect
  4. Apple review (~1-3 days) ‚Üí live

  ---
  OTA Updates (bonus)

  eas update
  Push JS bundle changes without App Store review. Works for everything that's
  not native code (UI changes, logic, translations). Instant delivery to users
  on next app open.
