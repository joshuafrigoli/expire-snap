● Not by default with Tailwind — but easy to set up. Two approaches:

  Option A: CSS custom properties (recommended)
  Define design tokens in index.css:
  :root {
    --color-primary: #22c55e;
    --color-danger: #ef4444;
    --color-warning: #f59e0b;
    --radius: 0.75rem;
  }
  Then use in Tailwind via tailwind.config.js:
  theme: {
    extend: {
      colors: {
        primary: 'var(--color-primary)',
        danger: 'var(--color-danger)',
      }
    }
  }
  Now bg-primary, text-danger etc. work everywhere. Change one var → whole app
  updates.

  Option B: Tailwind theme only (no CSS vars)
  Just extend the theme in tailwind.config.js with named colors. Simpler, but no
  runtime theme switching (e.g. dark mode toggle).

  Recommendation: Option A — gives you both Tailwind utility classes AND runtime
  theming (useful for dark mode later). One token change in :root updates
  everything.
